# Shannon vs Vigolium Agent Modes — Comparison

Comparison of Shannon (autonomous AI pentester by Keygraph) against Vigolium's Agent Swarm and Autopilot modes, specifically in source-aware (whitebox) scanning scenarios.

Shannon source code reference: https://github.com/keygraph/shannon

## Shannon Overview

Shannon is an autonomous white-box AI pentester. It analyzes source code to identify attack vectors, then executes real exploits to prove vulnerabilities. Only proven vulnerabilities with working PoCs are reported — zero false positives by design.

- **Language**: TypeScript / Node.js
- **Orchestration**: Temporal (durable workflows, queryable, auto-retry)
- **Agent SDK**: Claude Agent SDK (native multi-turn conversations)
- **Browser**: 5 isolated Playwright MCP instances
- **Editions**: Lite (AGPL-3.0) and Pro (commercial with SAST, SCA, secrets, business logic)

### Shannon Pipeline

```
Phase 1: Pre-Recon (source code analysis — 6 sub-tasks in parallel)
    ↓
Phase 2: Recon (attack surface mapping against live target)
    ↓
Phase 3-4: 5 Vulnerability + 5 Exploitation agents (parallel pipelines)
    - Each: vuln analysis → queue check → conditional exploit via Playwright/curl
    ↓
Phase 5: Reporting (assembles exploitation evidence into final report)
```

13 total agents: pre-recon, recon, 5x vuln specialists (injection, XSS, auth, SSRF, authz), 5x matching exploit agents, report.

## Part 1: Shannon vs Vigolium Agent Swarm (Source-Aware)

### Architecture & Orchestration

| Aspect | Shannon | Vigolium Swarm |
|---|---|---|
| **Language** | TypeScript / Node.js | Go |
| **Orchestration** | Temporal (durable workflows) | Custom phase runner with JSON checkpoints |
| **Agent SDK** | Claude Agent SDK (locked to Anthropic) | ACP — backend-agnostic (Claude, Gemini, OpenCode, custom CLI) |
| **Resume** | Git checkpoints + Temporal workflow resume + named workspaces | JSON checkpoint files + `--start-from <phase>` flag |
| **Parallelism** | 5 fixed parallel vuln→exploit pipelines | Batched master agent calls (>5 records), parallel source analysis sub-calls |

### Scanning Philosophy — The Fundamental Split

**Shannon is 100% agent-driven scanning.** No hardcoded scanner modules. LLM agents perform vulnerability analysis, craft payloads, and execute attacks via Playwright browsers or CLI tools. The agents ARE the scanner.

**Vigolium Swarm is a hybrid.** AI agents handle planning, source analysis, extension generation, and triage — but actual scanning is done by native Go modules (deterministic, fast, repeatable). The AI's job is to guide the native scanner: select modules, generate custom JS extensions for edge cases, and triage results.

- Shannon: **AI does the pentesting** (agents send requests, read responses, decide next steps)
- Vigolium: **AI directs the pentesting** (agents plan and review, Go code executes)

### Source Code Analysis

| Aspect | Shannon | Vigolium Swarm |
|---|---|---|
| **Approach** | Single Pre-Recon agent with 6 sub-tasks (architecture, endpoints, auth, XSS sinks, SSRF, data security) | 3 parallel LLM calls (explore → format + extensions) + native ast-grep SAST |
| **Output** | Freeform markdown deliverable consumed by downstream agents | Structured JSONL HTTP records + session config JSON + JS extensions |
| **Route extraction** | Agent reads code, documents endpoints in markdown | Agent extracts routes as parseable HTTP records; ast-grep does structural matching as backup |
| **Auth discovery** | Config-driven (user provides credentials in YAML) + code analysis for flow identification | Agent mines seed data, fixtures, `.env.example` for real credentials; auto-hydrates session tokens |
| **Credential handling** | User must provide credentials in config (form/SSO/API/basic + optional TOTP secret) | Agent autonomously discovers credentials from source code |

### Vulnerability Detection & Exploitation

| Aspect | Shannon | Vigolium Swarm |
|---|---|---|
| **Detection** | 5 specialist LLM agents (injection, XSS, auth, SSRF, authz) analyze code + recon data | Native Go modules (dozens of hardcoded checks) + AI-generated JS extensions for custom patterns |
| **Exploitation** | 5 LLM exploit agents send real attacks via Playwright/curl | Native scanner sends payloads; no separate exploitation phase |
| **Browser automation** | 5 isolated Playwright MCP instances for DOM XSS, auth flows | No browser — HTTP-level only |
| **PoC requirement** | Only reports exploited vulns with working PoCs | Reports findings from scanner matches; triage agent validates |
| **Custom payloads** | Agents craft payloads on-the-fly based on code analysis | Quick checks (declarative), snippets (JS body), full JS extensions — generated by plan agent |

### Triage & Validation

| Shannon | Vigolium Swarm |
|---|---|
| Exploitation IS validation — if exploit fails, not reported | Multi-round triage loop: agent reviews findings → confirms/rejects → optional rescan (up to 3 rounds) |
| Categories: EXPLOITED / BLOCKED_BY_SECURITY / OUT_OF_SCOPE / FALSE_POSITIVE | Categories: confirmed / false_positive / follow_up → rescan |
| Zero false positive rate by design | Batched triage (25 findings/batch), deduplication across extension variants |

### Shannon Advantages (vs Swarm)

1. **Browser-based exploitation** — Playwright enables DOM XSS, complex auth flows, SPA testing
2. **Proof-of-concept guarantee** — zero false positives (only exploited vulns reported)
3. **Durable orchestration** — Temporal provides production-grade workflow management vs custom checkpoints
4. **Deep agent autonomy** — agents iterate on failures, adapt payloads, chain multi-step attacks through conversation turns
5. **2FA/TOTP support** — built-in MCP tool generates TOTP codes during exploitation

### Vigolium Swarm Advantages (vs Shannon)

1. **Speed & cost** — native Go modules are orders of magnitude faster/cheaper than LLM-driven scanning
2. **Determinism** — Go modules produce repeatable results; LLM-only scanning varies between runs
3. **Backend agnostic** — works with Claude, Gemini, OpenCode; not locked to Anthropic
4. **Autonomous credential discovery** — mines source code for credentials instead of requiring user config
5. **Batched parallel planning** — handles large attack surfaces (>5 endpoints) with provenance tracking
6. **Extension ecosystem** — declarative quick_checks → snippets → full JS provide graduated complexity
7. **Native SAST integration** — ast-grep structural matching complements LLM analysis
8. **Scale** — scans thousands of endpoints efficiently (bottleneck is Go code, not LLM API calls)

## Part 2: Shannon vs Vigolium Autopilot (Source-Aware)

Autopilot is the Vigolium mode most architecturally similar to Shannon — both give an AI agent autonomous control over scanning.

### What's Similar

Both share the core idea: give an AI agent a sandbox and let it pentest autonomously.

| Shared Concept | Shannon | Vigolium Autopilot |
|---|---|---|
| Agent reads source code | Pre-recon agent analyzes codebase | Agent receives `{{.SourceCode}}` in prompt |
| Agent discovers routes | Entry Point Mapper sub-task | Agent reads framework patterns, runs `vigolium scan --only discovery` |
| Agent sends requests | Via Playwright MCP + curl | Via `vigolium scan-url` / `vigolium scan-request` commands |
| Agent reads responses | Direct from Playwright/curl output | Via `vigolium finding --json` / `vigolium traffic --json` |
| Agent iterates | Multi-turn conversation (maxTurns: 10,000) | Continuous terminal session (max 100 commands, 30min timeout) |
| Agent decides what to test next | Full autonomy per conversation turn | Full autonomy per command |
| Source-aware workflow | Code analysis → recon → vuln → exploit | Analyze routes → identify sinks → targeted scanning |

### Key Differences

#### 1. Scanning Execution Model

**Shannon**: Agent crafts and sends raw HTTP requests / browser interactions directly. It writes the payload, sends it, reads the response, decides if it worked. The agent IS the scanner.

**Vigolium Autopilot**: Agent invokes `vigolium scan-url <url> --module-tag sqli` — delegating to native Go scanner modules. The agent orchestrates which endpoints to scan with which modules, but payload generation and matching happens in Go code. The agent is a **scanner operator**, not the scanner itself.

Implication: Shannon agents can craft novel one-off payloads tailored to exact code patterns. Vigolium agents get speed/coverage of hardcoded modules but can't invent new payloads on the fly.

#### 2. Browser Automation

**Shannon**: 5 isolated Playwright MCP instances. Can fill login forms, handle JavaScript-rendered SPAs, execute DOM XSS, navigate multi-step auth flows, handle TOTP 2FA.

**Vigolium Autopilot**: HTTP-only. No browser. Cannot test DOM XSS, JS-heavy SPAs, or interactive auth flows requiring form submission. Agent can pass headers (`-H 'Authorization: ...'`) but can't do browser-based login.

#### 3. Agent Specialization

**Shannon**: 13 specialized agents with distinct roles. 5 vuln specialists + 5 matching exploit agents, each with a dedicated prompt. Runs in parallel.

**Vigolium Autopilot**: Single generalist agent with one prompt template (`autopilot-system.md`). Handles everything sequentially in one continuous session. No specialization.

#### 4. Exploitation vs Detection

**Shannon**: Explicit exploitation phase. Agents attempt real exploits, only report vulns with working PoCs. Categories: EXPLOITED / BLOCKED_BY_SECURITY / FALSE_POSITIVE.

**Vigolium Autopilot**: No explicit exploitation phase. Native scanner modules detect vulnerability indicators (response patterns, timing, errors) but don't necessarily prove full exploitation. No structured PoC generation.

#### 5. Durability & Recovery

**Shannon**: Temporal workflows — if process crashes mid-scan, Temporal replays from last checkpoint. Named workspaces enable parallel scans. Git checkpoints allow agent-level rollback.

**Vigolium Autopilot**: Session directory with output artifacts but no structured checkpoint/resume. 30-minute timeout / 100-command limit, then scan stops.

#### 6. Authentication

**Shannon**: Config-driven auth with form/SSO/API/basic support + TOTP MCP tool. User provides credentials in YAML. Agent navigates login flows via Playwright.

**Vigolium Autopilot**: Agent can discover credentials from source code (seed files, `.env.example`, fixtures). But auth execution limited to HTTP headers — no interactive login flows.

### Summary Matrix

| Capability | Shannon | Vigolium Autopilot |
|---|---|---|
| **Agent autonomy** | Full | Full |
| **Scanning engine** | LLM-driven (agent sends requests) | Native Go modules (agent invokes scanner) |
| **Browser automation** | Yes (5x Playwright) | No (HTTP-only) |
| **Agent count** | 13 specialized | 1 generalist |
| **Parallel execution** | 5 vuln + 5 exploit in parallel | Sequential (single session) |
| **Exploitation proof** | Required (PoC or not reported) | Not required (detection-based) |
| **Credential source** | User-provided config | Auto-discovered from source |
| **2FA support** | Yes (TOTP MCP tool) | No |
| **Durability** | Temporal workflows + git checkpoints | Session dir only, no resume |
| **Cost per scan** | High (all scanning is LLM inference) | Lower (LLM for orchestration, Go for scanning) |
| **Novel payload generation** | Yes (agent crafts payloads) | No (relies on module payloads) |
| **Backend lock-in** | Claude SDK only | Any ACP-compatible agent |

## Closing Gaps: What Vigolium Would Need to Match Shannon

To make Vigolium Autopilot fully equivalent to Shannon:

1. **Browser automation** — Playwright MCP or similar for DOM XSS, SPA testing, interactive auth
2. **Multi-agent specialization** — parallel vuln-class-specific agents instead of single generalist
3. **Structured exploitation with PoC generation** — prove vulnerabilities, not just detect indicators
4. **Durable workflow orchestration** — Temporal or equivalent for crash recovery and long-running scans
5. **2FA/TOTP support** — MCP tool or similar for generating auth codes during exploitation

Vigolium Swarm's existing advantages (speed, cost, determinism, scale, backend flexibility) are complementary — the ideal approach may combine Swarm's efficiency for broad coverage with Autopilot's autonomy for deep exploitation of high-value targets.
